## 2. `while` 處理程式碼之設計原理

實作 `while` 迴圈的設計核心在於 **「回跳機制」** 與 **「回填技術 (Backpatching)」**。

### A. 關鍵步驟解析
1.  **保存入口點 (`start_pc`)**：
    在解析 `while` 條件之前，必須先記住目前的指令地址（`quad_count`）。這是因為當迴圈內容執行完畢後，程式必須跳回這裡重新判斷條件是否成立。
2.  **條件跳轉 (`JMP_F`)**：
    解析 `while (expression)` 後，生成一個 `JMP_F` (Jump if False) 指令。此時我們還不知道迴圈主體 (Block) 有多長，因此跳轉目標暫時填入 `?`。
3.  **無條件回跳 (`JMP`)**：
    在迴圈主體的最後一條指令之後，生成一個無條件跳轉 `JMP`，目標指向第一步保存的 `start_pc`。
4.  **回填出口 (Backpatching)**：
    一旦主體解析完成，編譯器便獲得了迴圈結束後的正確地址（即當前的 `quad_count`）。此時回頭將第 2 步產生的 `JMP_F` 指令的結果欄位修改為該地址。

### B. 執行邏輯圖示
產生的中間碼 (Quadruples) 結構如下：

| PC | Op | Arg1 | Result | 說明 |
| :--- | :--- | :--- | :--- | :--- |
| **10** | (Condition) | - | t1 | **start_pc (入口)** |
| **12** | **JMP_F** | t1 | **16** | 條件不成立則跳出 |
| **13** | (Body...) | - | - | 迴圈內容 |
| **15** | **JMP** | - | **10** | **跳回起點重新判斷** |
| **16** | (Next) | - | - | 迴圈結束後的目標 |

## 3. 函數呼叫機制之運作原理

實作函數呼叫（Function Call）的設計核心在於 **「堆疊幀 (Stack Frame) 管理」** 與 **「環境隔離」**，這使得 P0 能夠支援遞迴呼叫。

### A. 實作流程
1.  **參數預留 (`PARAM`)**：在呼叫前，將參數值推入臨時堆疊。
2.  **建立堆疊幀 (`CALL`)**：遞增 `sp` (Stack Pointer)，開闢獨立的變數空間，並保存 `ret_pc` (返回位址) 與 `ret_var` (結果接收變數)。
3.  **環境恢復 (`RET_VAL`)**：執行 `return` 時，遞減 `sp` 銷毀當前幀，恢復執行流並回填結果。

### B. 虛擬機 (VM) 指令擴充
為了支援上述功能，VM 實作了以下關鍵操作碼：
* **JMP_F**：條件跳轉，用於跳出迴圈或跳過 `if` 分支。
* **JMP**：無條件跳轉，用於 `while` 結尾的回跳。
* **CALL / RET_VAL**：處理跨函數的執行跳轉與數值回傳。
